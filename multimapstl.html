Multimap STL

Multimap is similar to a map with the addition that multiple elements can have the same keys. Also, it is NOT required that the key-value and mapped value pair have to be unique in this case. One important thing to note about multimap is that multimap keeps all the keys in sorted order always. These properties of multimap make it very much useful in competitive programming.

** List of Functions of Multimap

  1. begin() – Returns an iterator to the first element in the multimap

  2. end() – Returns an iterator to the theoretical element that follows last element in the multimap

  3. size() – Returns the number of elements in the multimap

  4. max_size() – Returns the maximum number of elements that the multimap can hold

  5. empty() – Returns whether the multimap is empty

  6. pair<int,int> insert(keyvalue,multimapvalue) – Adds a new element to the multimap


List of Functions of Multimap

  1. multimap::operator=:-> It is used to assign new contents to the container by replacing the existing contents.

  2. multimap::crbegin() and multimap::crend() crbegin():-> returns a constant reverse iterator referring to the last element in the multimap container. crend() returns a constant reverse iterator pointing to the theoretical element before the first element in the multimap.

  3. multimap::emplace_hint():-> Insert the key and its element in the multimap container with a given hint.

  4. multimap clear():-> Removes all the elements from the multimap.

  5. multimap empty():-> Returns whether the multimap is empty.

  6. multimap maxsize():-> Returns the maximum number of elements a multimap container can hold.

  7. multimap value_comp():-> Returns the object that determines how the elements in the multimap are ordered (‘<‘ by default).
  8. multimap rend:-> Returns a reverse iterator pointing to the theoretical element preceding to the first element of the multimap container.

  9. multimap::cbegin():-> and multimap::cend() cbegin() returns a constant iterator referring to the first element in the multimap container. cend() returns a constant iterator pointing to the theoretical element that follows the last element in the multimap.

  10. multimap::swap():-> Swap the contents of one multimap with another multimap of same type and size.

  11. multimap rbegin:-> Returns an iterator pointing to the last element of the container.

  12. multimap size():-> Returns the number of elements in the multimap container.

  13. multimap::emplace():-> Inserts the key and its element in the multimap container.

  14. multimap::begin() and multimap::end() begin():-> returns an iterator referring to the first element in the multimap container. end() returns an iterator to the theoretical element that follows the last element in the multimap.

  15. multimap upper_bound():-> Returns an iterator to the first element that is equivalent to multimapped value with key-value ‘g’ or definitely will go after the element with key-value ‘g’ in the multimap.

  16. multimap::count():-> Returns the number of matches to element with key-value ‘g’ in the multimap.

  17. multimap::erase():-> Removes the key value from the multimap.

  18. multimap::find():-> Returns an iterator to the element with key-value ‘g’ in the multimap if found, else returns the iterator to end.

  19. multimap equal_range():-> Returns an iterator of pairs. The pair refers to the bounds of a range that includes all the elements in the container which have a key equivalent to k.

  20. multimap insert():-> Used to insert elements in the multimap container.

  21. multimap lower_bound():-> Returns an iterator to the first element that is equivalent to multimapped value with key-value ‘g’ or definitely will not go before the element with key-value ‘g’ in the multimap.

  22. multimap key_comp():-> Returns the object that determines how the elements in the multimap are ordered (‘<‘ by default).
